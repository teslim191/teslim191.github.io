CONTAINERS TECHNOLOGY: KUBERNETES AND DOCKER

1.0 Introduction

Container technology is an efficient and effective way to develop, test and deploy an application. They can be referred to as light weight OS-level virtualization that runs application and their dependencies in resource-isolated processes and ships it out as one unit (Spec India, 2019). The main purpose to execute software reliably from one computing environment to another. This can be from a developerâ€™s laptop to a test environment or from a staging environment into production. It can also be from a physical machine into a data centre to a virtual machine in a private or public cloud (Rubens, 2017). They are also an important part of IT security. Important public cloud computing providers such as Amazon Web Services, Microsoft Azure and Google Cloud Platforms all embrace and use container technology (DeMuro, 2019).

A single container can be used to run anything ranging from a micro service or software process to a larger application. Inside a container, you will find all the needed executable, binary code, libraries and configuration files, all coming together to form a single unit. Containers technology makes debugging very easy by freezing and restarting an exact copy of a system or application that you plan to deploy. Container packages are complete, small and efficient enough to execute an operation within seconds (Heusser, 2020).

Deciding to use a container technology or intending to expand its usage in a different area might prove challenging. The growth and expansion in container technology leads to a variety of choice such as deciding the standard to use, how to store the old version, methods of deploying new images and how to manage containers in production (Heusser, 2020).

Before containers gained popularity, virtual machines were used. Here, one physical server is used for multiple applications through a virtualization technology which is also referred to as virtual machine (DeMuro, 2019). Each virtual machine is made up of an entire operating system and an application to run. The physical server that runs these virtual machines each has its own operating system, with a single hypervisor emulation layer at the top. By running several operating system concurrently, there will be a lot of overhead on the server as resources get used, and the number of virtual machine is always limited.

In contrast, using a container technology, the server runs a single operating system, as each of the container can be able to share this single operating system with other container on the server. The parts of the operating system that are shared do not interfere with the other containers. This implies that when containers require less system resources of the server compared to virtual machines because they do not include operating system images. They are also significantly more efficient than virtual machines since a lot of containers can be packed into a single server. This allow application to be rapidly executed, patched and scaled. For example, while a virtual machine may require tons of gigabyte for storage, a container running similar program may only need megabytes.

Cloud computing providers such as Microsoft Azure and Google cloud platforms already embraced the use of container technologies such as Docker, Apache, Mesos, Kubernetes etc. Google and YouTube have also adopted it due to its simplicity, consistency, scalability and reliability (Spec India, 2019).
